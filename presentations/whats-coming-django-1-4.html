What's Coming in

Django 1.4

by Matthew Makai
Excella Consulting
Talk given to Django District on Feb. 21, 2012

What's Incompatible?

  • Python 2.4
  • "postgresql" type and PostgresSQL before 8.2

What's New?

  • ORM bulk create
  • Better WSGI support
  • Time zone support
  • In browser testing frameworks
What's

Incompatible?

Python 2.4

  • Django 1.4 drops Python 2.4
  • Context managers -> Python 2.5+
  • Python 2.5 - 2.7 required
  •  
  • No Python 3 support (yet)

PostgreSQL support

  • "postgresql" database type removed
  • use psycopg2 instead
  •  
  • PostgreSQL 8.2 minimum version
What's

New?

ORM bulk create

  • Efficient insertion for bulk data
  BlogPost.objects.bulk_create([

    BlogPost(title="Django 1.4 presentation"),

    BlogPost(title="Hello, Django District")

    BlogPost(title="First post!"),

  ])
  • save() is not called and no pre_save or post_save signals

Better WSGI support

  • startproject now creates wsgi.py
  • Dev server supports external WSGI configuration
  • Use deployment WSGI config in dev
  •  
  • mod_python removal in Django 1.5

Time zone support

  • Can enable time zone support
  • Store date/times as UTC
  • Localizes for users
  • In settings.py:
    USE_TZ = True
  • Recommended you install pytz

In browser testing

  • LiveServerTestCase
  • Launches live Django server
  • Integrate w/ in-browser frameworks
  • pip install selenium
  •  
  • Uses staticfiles contrib app

Other Highlights

  • QuerySet.prefetch_related()
  • Project layout & manage.py
  • HTML5 doctype on admin pages
  • Custom startproject templates
  •  
  • Admin no longer supports IE6
